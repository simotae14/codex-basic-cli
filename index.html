How I usually work.

The useful pattern is simple: define the goal, inspect context, implement carefully, validate results.

1

Understand the task

Clarify the real outcome, constraints, and what good looks like.

2

Inspect the context

Read the codebase, conventions, and existing patterns before changing anything.

3

Implement the right fix

Prefer clean architecture and root-cause solutions over quick patches.

4

Validate the result

Run focused checks where appropriate and leave the work in a solid state.
